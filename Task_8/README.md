# Task 8 - Database Security Understanding

## Objective
To understand database security risks by identifying SQL injection vulnerabilities and analyzing their impact.

## What I Did
- Identified injectable parameters in a vulnerable web application.
- Used SQLMap to test for SQL injection vulnerabilities.
- Extracted database names and tables.
- Analyzed exposure of sensitive user data.
- Documented the SQL injection attack flow.
- Studied the impact of database compromise.
- Suggested security fixes and best practices.

## Key Learnings
- How SQL injection affects database security.
- Importance of input validation and parameterized queries.
- Risks of improper database access control.
- Need for regular security testing.

## Tools Used
- SQLMap
- Vulnerable web application
- Web browser

## Outcome
This task improved my understanding of database security and common database attack techniques.
